# Architecture — 03 Tiny YOLO

## Folder tree

```
03-tiny-yolo/
├── README.md              # concept + 5-class rationale + workflow
├── ARCHITECTURE.md        # this file
├── IMPLEMENTATION_NOTES.md# why YOLOv8-nano, why these hyperparameters
├── requirements.txt       # ultralytics pin
├── dataset.yaml           # Ultralytics dataset spec: paths + class names
├── train.py               # fine-tune YOLOv8n from COCO weights
├── validate.py            # score best.pt on the held-out `test` split
└── data/
    └── README.md          # explains the YOLO directory layout we expect
```

Two artefacts are **created at runtime** and intentionally not in git:

```
runs/detect/<run_name>/
├── weights/best.pt        # the model validate.py loads
├── weights/last.pt
├── results.csv            # loss + mAP per epoch
└── *.png                  # confusion matrix, PR curves, sample preds
```

These live wherever you run `python train.py` from. The default
`runs/` folder is the Ultralytics convention. `.gitignore` should
exclude `runs/` and `data/synthetic_autosampler/` — both are
re-generable and bulky.

## Per-file responsibility

| File | Owns |
|---|---|
| [`dataset.yaml`](dataset.yaml) | Single source of truth for **where the data lives** and **what the classes are**. Both `train.py` and `validate.py` read it. Editing class names here without re-training will silently mis-label predictions, so this file pairs with `best.pt`. |
| [`train.py`](train.py) | Loads `yolov8n.pt` (COCO-pretrained nano), then calls `model.train()` with the data path, image size, batch size, and run name. No custom training loop — we delegate to Ultralytics. |
| [`validate.py`](validate.py) | Loads a trained `best.pt`, calls `model.val(split="test")`, prints overall mAP@0.5 / mAP@0.5:0.95 and a per-class breakdown, exits non-zero if the headline mAP@0.5 is below `MIN_MAP50 = 0.70`. |
| [`requirements.txt`](requirements.txt) | Pins `ultralytics>=8.2.0`. Torch / torchvision come in as transitive deps so the user does not have to pick a wheel by hand. |
| [`data/README.md`](data/README.md) | Documents the YOLO directory layout the scripts expect (`images/{train,val,test}/`, `labels/{train,val,test}/`). |

## How the files interact at runtime

```
                  dataset.yaml
                       │
                       │  read by both scripts
                       ▼
   ┌───────── train.py ─────────┐        ┌────── validate.py ──────┐
   │ YOLO("yolov8n.pt")          │        │ YOLO(<best.pt path>)    │
   │   .train(                    │        │   .val(                 │
   │     data=dataset.yaml,       │        │     data=dataset.yaml,  │
   │     epochs=..., imgsz=...,   │        │     split="test",       │
   │     batch=..., name=...,     │        │     imgsz=..., device=..│
   │   )                          │        │   ) -> metrics          │
   │                              │        │                         │
   │ writes runs/detect/<name>/   │  best.pt│ reads runs/detect/...   │
   │   weights/best.pt            │ ──────► │   weights/best.pt       │
   │   weights/last.pt            │        │ prints mAP report,      │
   │   results.csv + plots        │        │ exit 0 / 1              │
   └──────────────────────────────┘        └─────────────────────────┘
```

The handoff is **one file**: `best.pt`. Everything else is
configuration in `dataset.yaml`.

## Data layout the scripts expect

```
data/synthetic_autosampler/
├── images/
│   ├── train/<stem>.jpg
│   ├── val/<stem>.jpg
│   └── test/<stem>.jpg
└── labels/
    ├── train/<stem>.txt          # YOLO: class cx cy w h, normalised
    ├── val/<stem>.txt
    └── test/<stem>.txt
```

Stems must match between an image and its label file. Missing labels
are treated as "background-only image" — Ultralytics does not error.

## Dependencies (external to this folder)

- **`ultralytics`** (PyPI) — YOLOv8 model, trainer, evaluator.
  Brings in `torch`, `torchvision`, `numpy`, `pillow`, `opencv-python`
  as transitive deps.
- **A trained `best.pt`** for `validate.py` to load. Produced by
  `train.py`. Path defaulted to
  `runs/detect/autosampler_yolov8n/weights/best.pt`.
- **The dataset itself.** Not in git. Planned to be generated by
  exercise 5 (`auto-label-from-gazebo`).

## What this exercise does NOT touch

| Subsystem            | Where it lives                     |
|----------------------|------------------------------------|
| Gazebo world / camera | exercise 1 (SDF world)            |
| ROS 2 nodes / topics  | exercise 4 (live YOLO on camera)  |
| Auto-labelling        | exercise 5                         |
| MoveIt / arm motion   | exercises 18 – 22                  |

This exercise is pure ML — Python, PyTorch under the hood, no ROS.
That isolation is on purpose: it lets you iterate on the detector on
a workstation with a GPU, then deploy the resulting `best.pt` into
the ROS pipeline only once it passes the mAP bar.
